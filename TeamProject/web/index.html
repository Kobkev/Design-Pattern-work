Descripción del Sistema

El sistema de estudio y desarrollo que se elaboró en este proyecto tiene como enfoque el manejo de computadora (RAM, HDD, SDD, CPU, Teclados, Monitores etc). El sistema se encargará de mantener cuenta del inventario existente de manera ordenada mediante el uso de una base de datos mediante del uso del patron DAO y patron Singleton. Este permitirá a un administrador (cliente) mantener control sobre el manejo del inventario. Dado el hecho que podrá consultar la existencia de un artículo, agregar, modificar y eliminar registros que existan dentro del inventario. El sistema está diseñado con el fin de facilitar la manipulación del inventario con una interfaz controlador-intuitiva para el administrador. El desarrollo de éste está hecho haciendo uso de servlets en java junto con el uso de java web applications para llevarse a acabo. La idea principal detrás del proyecto es aplicar todos los conocimientos obtenidos durante el curso de Patrones de Desarrollo de software donde se realiza un enfoque a los Patrones GRASP (General Responsability Assignment Software), el cual se encarga de llevar prácticas de programación adecuadas con el fin de evitar las dependencias generales del programa o sistema desarrollado.
